О системе

Система собирает всю доступную информацию, обрабатывает и размещает в Навигаторе Знаний. Любая информация (из внешнего источника) размещается в подходящем по смыслу разделе. Информация созданная внутри системы, размещается в специальных системных разделах Навигатора.

Любая информация с чем-нибудь связана: владельцем, получателем, компаний, клиентом, другой информацией и т.д. Все эти связи автоматически определяются системой и фиксируются. Автоматическое определение связей задается с помощью правил. В будущем создание правил будет доступно пользователям.

Сообщения электронной почты, это одна из форм представления информации. Любое сообщение относится к определенной теме, система автоматически определит и разместит его в нужном разделе Навигатора. Если сообщение входит в цепочку, то все они будут автоматически связаны между собой.

Пользователи

Сотрудники компании работающие в системе.

Сотрудника создает Администратор компании.

Сотруднику может назначется роль в системе. Это может быть: Администратор, Эксперт, Пользователь.

Роли и права доступа

  • Роль Администратор

    • Имеет все права в системе.

  • Роль Эксперт

    • Дополнительно к возможностям роли Пользователь.

    Предназначена для управления не системными разделами Навигатора Знаний.

    Может создавать подразделы, создавать объекты в своих разделах, видеть всю информацию в них.

    Может создавать Кейсы. Это специальные объекты, которые описывают действия в той или иной ситуации пользователю системы. Пользователь системы может применять Кейсы.

  • Роль Пользователь

    • Основная роль Сотрудников в системе.

    Может подключать к системе аккаунты различных систем для сбора информации.

    Видеть свою информацию в Навигаторе, автоматически привязанную к его разделам.

    Использовать любые объекты созданные Экспертами.

Кейсы

  • Создание

    • Если в разделе Навигатора Знаний есть сообщения, вы можете создать Кейс на основе любого из них.

    Для этого откройте сообщение и нажмите "Создать кейс". Старайтесь использовать в поле "Запрос" слова наиболее близко характеризующие описании ситуации для которой будет применяться Кейс. Поле Алгоритм пока не используется, можно заполнить как угодно.

  • Применение

    • Для применения Кейсов в системе должно быть создано не менее 3 (трех) Кейсов в разных разделах Навигатора Знаний.

    После создания необходимо обучить систему применять Кейсы. Для этого в разделе Настройки: Навигатор Знаний нажмите "Обновить классификацию кейсов".

Сообщения и Кейсы

Это два основных способа представления информации в системе.

  • Сообщения

    • Все что получено через Аккаунты, созданные Пользователями системы.

    Сообщения нельзя редактировать, можно просматривать, привязывать к разделам Навигатора, использовать для создания Кейсов и применять к ним существующие Кейсы.

  • Кейсы

    • Это руководство к действию.

    Кейс создается Экспертом на основании какой-либо информации содержащийся в сообщении. Он содержит в себе описание проблемы, решение и алгоритм который будет использоваться при его применении.

    Когда в системе есть сообщение с информаций к которой можно применить тот или иной Кейс (т.е. описание проблемы в Кейсе очень похоже на содержание сообщения), об этом будет сообщено. Пользователь может использовать Кейс для ответа на сообщение. Решение будет использоваться как ответ.

Навигатора Знаний

Описывает области Знаний компаний. В разделы Навигатора помещается все доступная информация, автоматические распределенная системой.

Визуально представлена в виде дерева, но может быть преобразована в любой другой вид.

Вы можете найчить систему, какую информацию в какой раздел Навигатора Знаний разместить. Для этого свяжите любой сообщение, файл и т.д. с разделом Навигатора Знаний и проведите обучение системы в Настройках Навигатора Знаний.

Аккаунты интернет сервисов

Содержат данные для подключения к различным интернет сервисам. На текущий момент поддерживается только электронная почта, сервисы от Gmail и Yandex.

Подключение происходит по IMAP протоколу. По-умолчанию небходимо использовать порт: 993

Сообщения загружаются при входе Пользователя в систему и каждый раз при открытии главной страницы.

Пользователь может отключить аккаунт. Для этого необходимо в разделе Настройки: Аккаунты нажать на значок рядом с Аккаунтом.

Аккаунт не может быть удален. Это необходимо для сохранения информационных связей.

Отключение и включение пользователей

Администратор может отключить пользователя от системы. Для этого необходимо в разделе Настройки: Сотрудники и Аккаунты нажать на значок .
